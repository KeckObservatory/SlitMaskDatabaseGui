Submit a Mask

DEIMOS

A DEIMOS mask design file must be a multi-extension FITS file containing several tables which describe a desired slitmask. Such a FITS file will typically have been created by the DSIMULATOR program.

DEIMOS users take note: The e-mail addresses for the mask design author and the mask blueecho observer must exist as known users of this slitmask submission system. Please be sure that there are users with those e-mail addresses before using them to design a slitmask with DSIMULATOR.

LRIS

LRIS users may submit ASCII mask design files created by either of two means.

AUTOSLIT
This program is by Cohen, Shopbell, et al. It should be available from Caltech.
Note that neither Keck nor UCO/Lick has control nor detailed knowledge of the operation of AUTOSLIT.
The AUTOSLIT program produces multiple output files, and we are not aware of a canonical nomenclature scheme for distinguishing the various output files. There is also no clear indication of which file is which in the content of the headers of the output files from AUTOSLIT.
The output file that is needed for slitmask manufacture seems to be associated with a character string something like file3 in the setup for AUTOSLIT. This file is the one which contains multiple rows of mill coordinates for slitlets which are separated by lines containing the word newrow.
Please submit only this one kind of output from AUTOSLIT; i.e., the file containing the slitlet milling coordinates.
PREP (Simulator/Mapmask/Fabmask)
This program is by Phillips. It should be available from UCO/Lick.
The mill outputs from these LRIS programs will be converted automatically to multi-extension FITS files by this web interface, but they lack much of the metadata which simplifies DEIMOS data reduction.

LRIS users take note: The GUINAMEs which will be used for milling and selection of the masks are created using the first 8 characters of the file names used in the entry box below). Those 8 characters for the GUINAME for all masks must be unique, and duplicate names will be mangled in an attempt to ensure uniqueness. Until further notice the only control that LRIS users have over the GUINAME is to name the input files according to these conventions.

Submitting

Please use the file chooser below to tell your web browser the name(s) of your mask design file(s). Choose either DEIMOS or LRIS. Then press the Send button to have your browser transmit those file(s) to our web server. After that, follow the instructions on subsequent pages to insert the mask designs into the database for manufacturing.

Please take note of a change. Previously it was possible to submit tar files with more than one mask design. This is no longer allowed. Now it is possible to select as many as 10 files in the file chooser provided by the web browser. All of those selected files will be uploaded together.
