Slit Mask Help

Slitmask Options:

Help for the inventory options:
  • Archive

    • The ARCHIVE button will mark masks as ARCHIVED. The meaning of archived is that the mask is no longer READY for observing. If it was milled, the physics metal mask is no longer available for use. To move a mask design from ARCHIVED to READY, the PI will need to contact someone at Keck Observatory to REMILL the mask.


  • Remill

    • After being milled, masks are marked 'done' so that they do not pop up again in the 'to be milled' list. To make another copy of a mask that has been milled already, you will need to mark it 'unmilled' using the Remill button. NOTE: It is a Bad Idea to use the Remill button to re-use a slitmask from a previous run. The Observing Date will be past and the slitmask milling work queue will reject it. Instead we recommend you (a) regenerate the design file using DSIMULATOR or equivalent, and submit the revised version, or (b) edit the MDF (FITS file output from DSIMULATOR) and resubmit it with the corrected observing date. See Mask Dates Help for tips on this.


  • Plot

    • The Plot button is not dangerous. It produces a graphical representation of the mask design.

  • FITS file

    • The FITS file button is not dangerous. It generates a 'FITS chunk' for any mask design. This can take a while to generate and is not useful unless you already know what it is :-)


  • Detail

    • The Detail button is not dangerous. It shows all the data contained in the mask design and blueprint(s).
